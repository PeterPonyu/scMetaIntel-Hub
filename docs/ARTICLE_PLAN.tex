\section{Article Positioning}
% ============================================================================

\subsection{Central Claim}

We present \textbf{scMetaIntel-Hub}, a fully local, privacy-preserving
pipeline that converts natural language queries into grounded, cited answers
over single-cell genomics metadata --- without requiring gene expression
matrices or cloud API access. Through systematic benchmarking of 51 local
LLMs from 16 families (66 configurations including chain-of-thought
ablation) across 8 task dimensions on 2{,}189 GEO studies and 171 evaluation
queries, we show that mid-size local models (4--8B parameters) match or
exceed larger models on metadata-centric tasks when paired with hybrid
retrieval and organism filtering.

\subsection{Key Contributions}

\begin{enumerate}[leftmargin=2em]
  \item \textbf{Metadata-only intelligence}: A complete pipeline (parse
        $\to$ embed $\to$ retrieve $\to$ assemble $\to$ answer) that
        operates exclusively on GEO metadata and PubMed abstracts --- no
        expression matrices, no cloud dependencies.
  \item \textbf{Systematic local LLM benchmark}: 51 base models from 16
        architectural families in 66 configurations (including
        chain-of-thought ablation) evaluated across 8 domain-specific tasks,
        using a curated corpus of 2{,}189 enriched GEO studies and 171
        evaluation queries.
  \item \textbf{Empirically validated pipeline recipe}: Hybrid retrieval
        with organism filtering outperforms neural reranking at half the
        latency; a 4B model (qwen3.5-4b) statistically ties the top 32B
        model (aya-expanse-32b) at composite 0.779; 4-bit quantisation
        incurs negligible quality loss.
  \item \textbf{Curated evaluation resource}: 2{,}189 enriched GSE metadata
        files spanning 42 organisms, 171 queries across 7 categories and
        3 difficulty levels, and fully reproducible benchmark scripts.
\end{enumerate}

\subsection{Target Venue}

\begin{itemize}
  \item Primary: Springer Nature journal (sn-nature template)
  \item Alternative: \textit{Bioinformatics} (Oxford),
        \textit{Nucleic Acids Research}, \textit{Briefings in Bioinformatics}
\end{itemize}

  \item Search gap: keyword-only interfaces, no semantic reasoning.

\subsection{Section 2: Methods (3--4 pages)}

\subsubsection{2.1 Data Collection and Curation}

\begin{itemize}
  \item 2{,}189 GSE accessions enriched from GEO SOFT + PubMed E-utilities.
  \item 5-stage cleaning pipeline (text filtering, cell-line removal,
        disease migration, non-disease removal, text gap-filling).
  \item Coverage: 42 organisms, 83.8\% tissue, 44.8\% disease, 72.6\%
        cell type, 69.0\% PubMed, 100\% document text.
  \item Table~1: Corpus statistics.
\end{itemize}

\subsubsection{2.2 Pipeline Architecture}

\begin{itemize}
  \item Figure~1: 5-stage pipeline diagram.
  \item All local: Ollama (LLMs), HuggingFace (embeddings/rerankers),
        Qdrant (vector store).
  \item Hardware: RTX~5090 Laptop (24.5\,GB VRAM), CUDA~13.0, Linux~6.14.
\end{itemize}

\subsubsection{2.3 Evaluation Queries}

\begin{itemize}
  \item 171 queries across 7 categories: basic~(35), disease~(33),
        multi\_constraint~(31), cell\_type~(29), modality~(18),
        natural\_language~(13), organism~(12).
  \item 3 difficulty levels: easy~(48), medium~(83), hard~(40).
  \item 540 unique expected GSEs (24.7\% corpus coverage).
\end{itemize}

\subsubsection{2.4 Benchmark Design (8 Tasks)}

\begin{enumerate}
  \item[A.] Query Parsing (exact match, $n=163$)
  \item[B.] Metadata Extraction (per-field F1, $n=1{,}828$)
  \item[C.] Ontology Normalisation (F1, variable $n$)
  \item[D.] Answer Generation (citation P/R/grounding, $n=171$)
  \item[E.] Inference Speed (tok/s)
  \item[F.] Relevance Judgment (F1, $n=1{,}340$)
  \item[G.] Domain Classification (accuracy, $n=1{,}649$)
  \item[H.] Organism + Modality (accuracy/F1, $n=2{,}106$)
\end{enumerate}

Composite score: weighted combination of all 8 tasks (Eq.~1).

\subsubsection{2.5 Model Configurations}

51 base models from 16 families (Qwen, Gemma, Llama, Mistral, Phi,
DeepSeek, Granite, Falcon, Aya, GLM, InternLM, Yi, Solar, EXAONE,
StarCoder, Command-R), spanning 0.5B--32B parameters. Chain-of-thought
ablation adds 15 think-mode variants, yielding 66 total configurations.
Four Q4/Q8 quantisation pairs for ablation.

% ---------------------------------------------------------------------------
\subsection{Section 3: Results (3--4 pages)}

\subsubsection{3.1 Embedding Model Comparison}
14 models (10 general, 4 biomedical). mxbai-embed-large wins retrieval
(R@50=0.457); biomedical models lead ontology matching. Figure~2.

\subsubsection{3.2 Retrieval Strategy Comparison}
6 strategies. hybrid+filter optimal (R@50=0.478, 86\,ms). Reranking
doubles latency with no recall gain. Table~2.

\subsubsection{3.3 LLM Benchmark Results}
66 configurations. llama3.1-8b-q4 ranks first (composite 0.776). Scale
does not predict performance ($\rho=-0.08$). Citation recall is the most
discriminating metric (range 0.000--1.000). Figure~3 (heatmap), Figure~4
(scaling law), Table~3 (top-5).

\subsubsection{3.4 Context Optimisation}
240 configurations. Structured format, $k=3$, concise prompt is optimal.
Figure~5.

\subsubsection{3.5 End-to-End Pipeline}
4 pipeline configs. Optimised-fast (mxbai + hybrid+filter + qwen2.5-1.5b)
achieves best quality--latency trade-off. Figure~6.

% ---------------------------------------------------------------------------
\subsection{Section 4: Discussion (1.5 pages)}

\begin{enumerate}[leftmargin=2em]
  \item Task-specific model routing: no single model dominates all tasks.
  \item Reranking paradox: general rerankers degrade domain retrieval.
  \item Quantisation as free lunch: Q4 $\approx$ Q8 quality, 1.4$\times$ speed.
  \item Metadata quality as performance ceiling.
  \item Limitations: GEO-only, English-only, metadata-only, single-GPU,
        no cloud model comparison.
\end{enumerate}

% ---------------------------------------------------------------------------
\subsection{Section 5: Conclusion (0.5 page)}

Optimal recipe + resource release (2{,}189 GSEs, 171 queries, all code).
